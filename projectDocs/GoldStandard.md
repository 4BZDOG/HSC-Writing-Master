# Universal Gold Standard Guidelines for HSC Question Generation (v2.2.2)

## I. The Six Pillars Architecture
Every high-quality HSC question generated by this system is built on six interdependent elements. In **v2.2.1**, the `PromptGeneratorModal` and `ContentAuditModal` automate the alignment of these pillars.

1.  **SYLLABUS ITEM**: Defines the content boundaries.
2.  **COURSE OUTCOMES**: Defines the skills being assessed.
3.  **SCENARIO**: Provides the authentic context (WHO + WHAT + WHY).
4.  **KEYWORDS**: Identifies the sophisticated vocabulary needed (Tier 1/2/3).
5.  **MARKING CRITERIA**: Describes quality levels of skill application (Descending Order).
6.  **SAMPLE ANSWERS**: Demonstrates what quality looks like in practice.

**Critical Principle**: The AI is instructed that each element validates the others. A keyword listed must appear in the sample answer. A criteria point must be evidenced in the sample answer.

## II. Cognitive Demand & Verbs
The system strictly enforces the **NESA Command Verb Hierarchy**.
If the syllabus outcome uses the verb "Describe", the generator will rarely suggest an "Evaluate" question unless explicitly overridden, to ensure alignment with the scope of study.

### The "Including" Rule
When a syllabus item says: *"Explain X, including A, B, and C"*
The **Content Audit Studio** logic checks for coverage of all sub-items (A, B, and C) across the prompt dataset for that dot point.

## III. Scenario Generation: The Authenticity Engine
The `gemini-3-flash-preview` model is used to generate scenarios following the formula:
**WHO + WHAT + WHY (Consequence)**

*   **Constraint Injection**: The system now injects constraints (Temporal, Financial, Ethical) to prevent generic textbook answers.
*   *Example*: Instead of "Discuss security", the system generates "Discuss the ethical tension of implementing facial recognition given a limited budget and privacy concerns."

## IV. Keywords: The Sophistication Markers
The `KeywordEditor` component manages three tiers of vocabulary:

*   **Tier 1 (Syllabus)**: Core terms (e.g., "Encryption", "Metaphor").
*   **Tier 2 (Academic)**: Analysis terms (e.g., "Consequently", "Implies", "Juxtaposes").
*   **Tier 3 (Contextual)**: Scenario specific (e.g., "GDPR Compliance", "Patient confidentiality").

**The 70% Rule**: The automated Quality Check (`QualityCheckModal`) verifies that at least 70% of the listed keywords appear in the Band 6 Sample Answer.

## V. Marking Criteria: The Quality Ladder
The system generates rubrics using the **Descending Quality Descriptor** pattern:
*   **Band 6**: "Effectively", "Insightfully", "Comprehensively".
*   **Band 5**: "Clearly", "Soundly".
*   **Band 4**: "Adequately", "Generally".
*   **Band 3**: "Basic", "Simple".
*   **Band 2**: "Limited", "Superficial".
*   **Band 1**: "Fragmented", "Minimal".

## VI. Sample Answers
Sample answers are generated using `gemini-3-pro-preview` with a specific `thinkingBudget`.
*   **Structure**: PEEL (Point, Evidence, Explanation, Link) is enforced for 4+ mark questions.
*   **Length**: Tuned to approx. 35-45 words per mark.
*   **Differentiation**: The "Generate Sample Answer" tool can explicitly create a "Band 4" response (containing common errors) versus a "Band 6" response (exemplar) to help students see the gap.
